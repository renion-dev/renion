# Інструкція для користувачів RENION — Opportunity Hunter

**Версія:** v1.0.0  
**Дата:** 2026-07-21  

---

## Про що цей документ?

Цей посібник допоможе вам зрозуміти, як працює **RENION Opportunity Hunter**, як запускати сканування, переглядати гіпотези та використовувати всі можливості системи.

---

## 1. Що таке RENION Opportunity Hunter?

**RENION** — це операційне середовище для автономних агентів.  
**Opportunity Hunter** — перший модуль, який автоматично:

- Збирає дані з 12+ джерел (Hacker News, Reddit, GitHub Issues, вакансії).
- Аналізує їх через локальну AI-модель.
- Виявляє повторювані проблеми та формує гіпотези для бізнесу.
- Оцінює ринковий потенціал (TAM/SAM/SOM).
- Генерує сучасні лендинги для кожної гіпотези.
- Створює інвойси для оплати.

---

## 2. Як почати?

### Для кінцевих користувачів (через веб-інтерфейс)

1. Відкрийте веб-інтерфейс за адресою, де розгорнуто RENION (наприклад, `http://localhost:8000` або публічний URL).
2. Ви побачите головну сторінку з описом проєкту.
3. Перейдіть на сторінку **"Гіпотези"** (кнопка `🚀 Переглянути гіпотези`).
4. Натисніть кнопку **"Запустити сканування"** — система почне збір та аналіз даних.
5. Зачекайте 2–5 хвилин (статус оновлюватиметься автоматично).
6. Після завершення з’являться нові гіпотези в таблиці.

### Для адміністраторів (локальне встановлення)

Якщо ви хочете запустити RENION на власному сервері або комп’ютері:

```bash
# 1. Клонуйте репозиторій
git clone <repo-url>
cd renion

# 2. Створіть віртуальне середовище
python -m venv venv
source venv/bin/activate  # Linux/macOS
venv\Scripts\activate     # Windows

# 3. Встановіть залежності
pip install -e .

# 4. Налаштуйте .env (скопіюйте з .env.example)
cp .env.example .env

# 5. Запустіть сканування через CLI (опціонально)
python -m src.interfaces.cli

# 6. Запустіть веб-сервер
uvicorn src.interfaces.web:app --host 0.0.0.0 --port 8000 --reload
3. Робота з веб-інтерфейсом
Головна сторінка (/)

    Презентація проєкту RENION.

    Кнопка для переходу до списку гіпотез.

Сторінка гіпотез (/hypotheses)

    Таблиця з усіма згенерованими гіпотезами.

    Кнопка "Запустити сканування" — запускає новий збір даних.

    Статус сканування — відображає поточний стан (не запущено / виконується / завершено / помилка).

    Колонки таблиці:

        # — номер гіпотези.

        Problem — опис проблеми.

        Headline — заголовок для лендингу.

        MVP — мінімальний життєздатний продукт.

        Created — дата створення.

        Landing — чи згенеровано лендинг.

        Action — посилання на детальний перегляд та лендинг.

Сторінка детального перегляду гіпотези (/hypothesis/{id})

    Повна інформація про гіпотезу:

        Проблема, MVP, гіпотеза, частота згадувань, CTA.

        Оцінка ринку: TAM (Total Addressable Market), SAM, SOM.

        Посилання на згенерований лендинг.

    Тут ви можете побачити, наскільки перспективною є гіпотеза.

Лендинг гіпотези (/landing/{id})

    Сучасна HTML-сторінка, створена автоматично.

    Містить:

        Заголовок (Headline).

        Опис проблеми.

        Опис MVP.

        Кнопку "Pay Now" для оплати доступу (сума $1.69).

    Лендинг можна використовувати для збору email або продажу.

4. Оплата та інвойси

    Кожна гіпотеза автоматично отримує інвойс на суму $1.69.

    Оплата проводиться через SimulatedProvider (заглушка) або Stripe (якщо налаштовано).

    Після успішної оплати статус інвойсу змінюється на paid, а платіж зберігається в базі.

    У майбутньому планується підписка Free/Pro/Business.

5. Розуміння TAM/SAM/SOM

    TAM (Total Addressable Market) — загальний обсяг ринку в доларах.

    SAM (Serviceable Addressable Market) — частина ринку, яку ви можете охопити.

    SOM (Serviceable Obtainable Market) — реально досяжна частка.

    Ці оцінки генеруються AI на основі аналізу даних та частоти згадувань проблеми.

    Вони допомагають пріоритизувати гіпотези за потенційним доходом.

6. Часті запитання (FAQ)

Q: Сканування триває довго. Це нормально?
A: Так, зазвичай 2–5 хвилин. Якщо більше — перевірте з’єднання з інтернетом та наявність Ollama.

Q: Де зберігаються дані?
A: У файлі aion.db (SQLite). Лендинги — у папці landings/.

Q: Як оновити гіпотези?
A: Натисніть "Запустити сканування" — старі гіпотези залишаться, нові додадуться.

Q: Чи можу я отримати доступ до API?
A: Так, є ендпоїнти /api/scan, /api/scan/status, /api/pay/{id}. Документація буде додана пізніше.

Q: Як додати власні джерела даних?
A: Відредагуйте src/config.py — додайте URL у RSS_SOURCES, GITHUB_REPOS або JOB_RSS_SOURCES.

Q: Як використовувати реальний Twitter постинг?
A: Додайте Twitter API ключі у .env, і система автоматично публікуватиме гіпотези.

Q: Чи можна використовувати Stripe?
A: Так, змініть PAYMENT_PROVIDER=stripe у .env та додайте Stripe ключі.
