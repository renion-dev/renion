# ADR-009: Запуск реклами (заглушка з логуванням)

**Дата:** 2026-07-21  
**Статус:** Прийнято  

## Контекст
Після генерації гіпотези та створення лендингу, наступним логічним кроком є запуск реклами для валідації попиту. Це дозволяє:
- Перевірити гіпотезу на реальній аудиторії
- Зібрати перші конверсії (email, кліки)
- Отримати дані для прийняття рішення

На поточному етапі ми не маємо інтеграції з рекламними платформами (Facebook Ads, Google Ads), тому робимо заглушку.

## Рішення

### Підхід
- Створити модуль `src/application/advertising.py` з класом `AdvertisingManager`.
- Додати метод `launch_campaign(hypothesis_id, hypothesis_data)`, який:
  - Логує намір запустити рекламу (з даними гіпотези)
  - Зберігає запис про запуск у БД (об'єкт типу `AdCampaign`)
  - Публікує подію `ad_campaign_launched`
  - Повертає статус "simulated" з імітованим ID кампанії
- У майбутньому замінити заглушку на реальні API.

### Дані для логування
- hypothesis_id
- landing_url (шлях до лендингу)
- target_audience (згенеровано на основі проблеми)
- budget (фіксований, наприклад $100)
- platform (Facebook, Google тощо — поки "simulated")

### Інтеграція
- Підписати обробник на подію `hypothesis_generated` (вже є).
- Після генерації лендингу автоматично запускати рекламу.

## Альтернативи
- Інтеграція з Facebook Ads API одразу — потребує OAuth, ключі, складніше.
- Ручний запуск — не автоматизує процес.

## Наслідки
- Система закриває повний цикл: проблема → гіпотеза → лендинг → реклама.
- Можна відстежувати, скільки гіпотез "пішли в рекламу".
- У майбутньому легко замінити заглушку на реальну інтеграцію.

## Майбутні плани
- Інтеграція з Facebook Ads API (з використанням `facebook-business` SDK).
- Інтеграція з Google Ads API.
- Збір статистики (покази, кліки, конверсії).
- Автоматичне прийняття рішень на основі даних (продовжити/зупинити кампанію).
