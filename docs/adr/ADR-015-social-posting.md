# ADR-015: Автоматичний соціальний постинг (безкоштовна реклама)

**Дата:** 2026-07-21  
**Статус:** Прийнято  

## Контекст
Після генерації гіпотези та лендингу необхідно залучити перших користувачів для валідації. Платна реклама (Facebook/Google Ads) потребує бюджету та налаштувань. На першому етапі доцільно використовувати безкоштовні органічні канали для отримання першого трафіку та зворотного зв'язку.

## Рішення

### Канали для автоматичного постингу

1. **Twitter/X** — через API (безкоштовно до 1500 твітів/місяць).
2. **Reddit** — через API (безкоштовно, з дотриманням правил сабредітів).
3. **LinkedIn** — через API (обмежено, або генерація тексту для ручного копіювання).
4. **Hacker News** — через офіційний API (безкоштовно, "Show HN").
5. **Email-розсилка** — поки що заглушка (збір email з лендингу).

### Технічна реалізація

- Модуль `src/application/social_poster.py` з класом `SocialPoster`.
- Для кожної платформи — окремий метод.
- Генерація тексту через LLM (Ollama) з урахуванням специфіки платформи.
- Зберігання історії постів у БД (об'єкт `SocialPost`).

### Налаштування

- API ключі через змінні середовища (Twitter, Reddit тощо).
- Якщо API ключі відсутні — генерувати текст і виводити в консоль для ручного копіювання.

### Інтеграція

- Підписка на подію `hypothesis_generated`.
- Автоматичний запуск постингу після генерації гіпотези та лендингу.
- Логування результатів.

## Альтернативи
- Платна реклама (Facebook/Google) — потребує бюджету.
- Ручний постинг — не масштабується.

## Наслідки
- Безкоштовний трафік на лендинги.
- Можливість валідувати гіпотези без витрат.
- Потрібні API ключі для Twitter, Reddit (можна отримати безкоштовно).

## Майбутні плани
- Інтеграція з Buffer/Hootsuite для керування постами.
- Платна реклама як додатковий канал після валідації.
- Аналітика ефективності постів (кліки, конверсії).
