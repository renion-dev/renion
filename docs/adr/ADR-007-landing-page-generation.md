# ADR-007: Генерація лендингу для гіпотез

**Дата:** 2026-07-21  
**Статус:** Прийнято  

## Контекст
Після того як Opportunity Hunter згенерував гіпотези (проблема + MVP + CTA), наступним логічним кроком є створення лендингу для валідації попиту. Лендинг дозволяє:
- Зібрати email-адреси зацікавлених користувачів
- Перевірити гіпотезу без написання коду
- Отримати зворотний зв'язок

## Рішення

### Підхід
- Використовувати LLM (Ollama) для генерації HTML-коду лендингу на основі гіпотези.
- Зберігати згенерований HTML у файл `landings/{hypothesis_id}.html`.
- Лендинг має містити:
  - Заголовок (landing_headline)
  - Опис проблеми (description)
  - Опис MVP (mvp)
  - Заклик до дії (cta) — кнопка "Залишити email"
  - Просту форму для збору email (без бекенду, поки що статичний HTML)

### Технічна реалізація
- Створити модуль `src/application/landing_generator.py` з класом `LandingGenerator`.
- Використовувати метод `generate_landing(hypothesis_data)`.
- Промпт для LLM: описати структуру лендингу, використовувати сучасний CSS (темний/світлий дизайн).
- Якщо генерація не вдається — використовувати шаблон-заглушку.

### Альтернативи
- Використання готових шаблонів (HTML/CSS) — менш гнучко, але швидше.
- Інтеграція з Webflow/тильдою — зайва залежність.

### Наслідки
- Користувач отримує готовий лендинг для кожної гіпотези.
- Можна поширювати посилання на лендинг для збору попиту.
- У майбутньому можна додати автоматичний деплой на Vercel/Netlify.

## Майбутні плани
- Підключити збір email через простий бекенд (FastAPI + SQLite).
- Додати A/B тестування різних версій лендингу.
- Автоматичний запуск реклами через Facebook/Google Ads.
