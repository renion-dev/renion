# ADR-011: Економічний шар (абстракція платежів)

**Дата:** 2026-07-21  
**Статус:** Прийнято  

## Контекст
AION як операційне середовище для автономних агентів повинен мати вбудований економічний шар. Кожна дія агента має оцінюватися за вартістю/цінністю, а система повинна підтримувати платежі, інвойси та транзакції.

Важливо: ядро не повинно знати про конкретні платіжні системи (Stripe, PayPal тощо). Це порушує принцип "Business Logic Never Lives in Kernel".

## Рішення

### Доменні сутності (ядро)
- **Payment** — платіж (сума, валюта, статус, метод, timestamp)
- **Transaction** — транзакція (пов'язана з Payment, має референс до зовнішнього провайдера)
- **Invoice** — рахунок (сума, валюта, статус, посилання на об'єкт, за який платять)

### Абстракція провайдера
- `PaymentProvider` — інтерфейс для платіжних систем
  - `create_payment(amount, currency, metadata) -> Payment`
  - `confirm_payment(payment_id) -> bool`
  - `refund_payment(payment_id) -> bool`
  - `get_status(payment_id) -> str`

### Use cases (application)
- `ProcessPayment` — обробка платежу (викликає провайдера)
- `CreateInvoice` — створення рахунку
- `VerifyPayment` — перевірка статусу платежу

### Інфраструктура (реалізація)
- `StripeProvider` — реалізація через Stripe API
- `PayPalProvider` — реалізація через PayPal API
- `SimulatedProvider` — для тестування (заглушка)

### Інтеграція з Opportunity Hunter
- Після генерації гіпотези та лендингу створюється `Invoice`
- На лендингу з'являється кнопка "Оплатити"
- Оплата → створення `Payment` → оновлення статусу

## Альтернативи
- Пряма інтеграція з Stripe в коді лендингу — порушує архітектуру.
- Використання готового платіжного шлюзу (наприклад, Paddle) — зайва залежність.

## Наслідки
- Ядро залишається чистим і незалежним від зовнішніх сервісів.
- Легко додавати нові платіжні системи (зміна тільки інфраструктурного шару).
- Можна тестувати без реальних грошей (SimulatedProvider).

## Майбутні плани
- Реалізація StripeProvider.
- Додавання webhook для обробки підтверджень.
- Підтримка підписок (recurring payments).
