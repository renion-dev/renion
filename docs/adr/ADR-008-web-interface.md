# ADR-008: Веб-інтерфейс для перегляду гіпотез та лендингів

**Дата:** 2026-07-21  
**Статус:** Прийнято  

## Контекст
На поточному етапі Opportunity Hunter працює через CLI і зберігає результати в SQLite. Для зручного перегляду гіпотез і лендингів необхідний веб-інтерфейс.

## Рішення

### Підхід
- Використовувати **FastAPI** (легкий, асинхронний, вбудована документація).
- Інтерфейс буде читати дані з тієї ж бази `aion.db`.
- Сторінки:
  - **Список гіпотез** (таблиця з ID, проблемою, датою, посиланням на лендинг).
  - **Детальний перегляд** гіпотези з усіма полями.
  - **Посилання на згенерований лендинг** (статичний HTML з папки `landings/`).
- Статичні файли (лендинги) будуть доступні за маршрутом `/landings/{filename}`.

### Технічна реалізація
- Додати залежності: `fastapi`, `uvicorn`.
- Створити модуль `src/interfaces/web.py` з FastAPI додатком.
- Використовувати `jinja2` для шаблонів (опціонально, можна простий HTML).
- Для запуску: `uvicorn src.interfaces.web:app --reload`.

### Альтернативи
- Використання Flask — більш громіздкий.
- Простий HTTP-сервер без API — незручно.
- Streamlit — не підходить для кастомного дизайну.

### Наслідки
- Користувач отримує зручний доступ до результатів.
- Можна додати фільтри, пошук, експорт.
- У майбутньому можна додати API для інтеграцій.
