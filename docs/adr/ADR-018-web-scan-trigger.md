# ADR-018: Веб-запуск сканування

**Дата:** 2026-07-21  
**Статус:** Прийнято  

## Контекст
На поточному етапі сканування запускається тільки через CLI (`python -m src.interfaces.cli`). Це ускладнює використання продукту для кінцевих користувачів. Необхідно додати можливість запускати сканування безпосередньо з веб-інтерфейсу.

## Рішення

### Підхід
- Додати кнопку "Запустити сканування" на головній сторінці.
- При натисканні кнопки надсилається POST-запит на `/api/scan`.
- Сканування запускається у фоновому режимі (asyncio.create_task).
- Статус сканування зберігається в БД як об'єкт `ScanJob`.
- Користувач бачить статус сканування на головній сторінці (idle, running, completed, failed).

### Технічна реалізація
- **Доменний об'єкт `ScanJob`** — зберігає статус, час початку/завершення, кількість гіпотез, помилки.
- **Методи Storage** для збереження/отримання ScanJob.
- **Ендпоїнт `/api/scan`** — запускає сканування (якщо не запущено).
- **Ендпоїнт `/api/scan/status`** — повертає поточний статус.
- **Оновлення головної сторінки** — додає кнопку та індикатор статусу.

### Обмеження
- Сканування може тривати 5-10 хвилин, тому важливо показувати статус.
- Зараз тільки одне сканування одночасно (запобігання повторним запускам).

### Альтернативи
- Використання WebSocket для оновлення статусу в реальному часі — зайва складність для MVP.
- Запуск через окремий процес (subprocess) — ускладнює управління.

## Наслідки
- Користувач отримує зручний спосіб запуску сканування.
- Зменшується поріг входу — не потрібно використовувати CLI.
- У майбутньому можна додати автоматичне сканування за розкладом.

## Майбутні плани
- Додати WebSocket для оновлення статусу в реальному часі.
- Додати автоматичне сканування (cron / scheduler).
- Додати історію сканувань.
