# ADR-003: Повноцінний Opportunity Hunter з LLM (Ollama) та розширеними джерелами

**Дата:** 2026-07-21  
**Статус:** Прийнято  

## Контекст
Opportunity Hunter має перетворитися з простого RSS-парсера на автономного агента, який:
- Аналізує десятки джерел (Reddit, GitHub Issues, Product Hunt, Hacker News, форуми, вакансії, відгуки)
- Виявляє незадоволений попит (проблеми, про які скаржаться багато людей)
- Генерує MVP (мінімально життєздатний продукт) у вигляді концепції
- Створює лендинг для збору попиту
- Запускає рекламу (заглушка на першому етапі)
- Відстежує оплати (поки що вручну або через симуляцію)

## Рішення

### Джерела даних (розширений список)
1. **Reddit** — через RSS (додаємо `.rss` до URL) для сабредітів: r/startups, r/SideProject, r/Entrepreneur, r/growmybusiness
2. **GitHub Issues** — через RSS або GitHub API (для публічних репозиторіїв)
3. **Product Hunt** — через RSS (https://www.producthunt.com/feed)
4. **Hacker News** — вже є
5. **Вакансії** — через RSS з дощок вакансій (наприклад, Stack Overflow Jobs)
6. **Негативні відгуки** — через RSS з сайтів відгуків (наприклад, Trustpilot)

### LLM (локально через Ollama)
- Використовуємо модель `llama3` (або `mistral`) через API Ollama (http://localhost:11434)
- Аналіз текстів: виділення проблем, болю, запитів
- Кластерізація скарг за темами
- Генерація гіпотез у вигляді: "Проблема X -> Рішення Y -> MVP Z"

### Генерація MVP та лендингу
- **MVP** — текстове описання мінімального продукту (згенероване LLM)
- **Лендинг** — генерація HTML/CSS коду (з використанням шаблонів та LLM)

### Запуск реклами (перша ітерація — заглушка)
- На першому етапі — просто логування наміру запустити рекламу (без реальних API)
- У майбутньому — інтеграція з Facebook/Google Ads

## Компроміси
- Використання RSS замість API — простіше, але менше даних (без коментарів, оцінок)
- Генерація лендингу — базова, без інтерактивності
- Запуск реклами — поки тільки концепція

## Наслідки
- Система стає набагато кориснішою, але складнішою.
- Потрібна потужна локальна машина для Ollama (або зменшення моделі).
- Додається багато нових залежностей та конфігурацій.

## Майбутні плани
- Інтеграція з Facebook Ads API та Google Ads.
- Додавання веб-інтерфейсу для перегляду гіпотез.
- Автоматичний збір оплат через Stripe.
