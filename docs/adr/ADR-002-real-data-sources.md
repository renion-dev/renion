# ADR-002: Підключення реальних джерел даних (RSS/Atom)

**Дата:** 2026-07-21  
**Статус:** Прийнято  

## Контекст
На поточному етапі Opportunity Hunter використовує заглушку для RSS. Для отримання реальної цінності необхідно підключити справжні джерела даних.

## Рішення

### Джерела даних (перша черга)
1. **Hacker News RSS** — `https://news.ycombinator.com/rss`
2. **Reddit (r/startups)** — `https://www.reddit.com/r/startups/.rss`
3. **TechCrunch** — `https://feeds.feedburner.com/TechCrunch`

### Технічна реалізація
- Використовуємо стандартну бібліотеку `feedparser` для парсингу RSS/Atom.
- Конфігурація: список URL зберігається у змінних середовища або файлі `config.py`.
- Кожне джерело сканується асинхронно з обмеженням на частоту запитів.
- При помилках (таймаут, 404) — логування та перехід до наступного джерела.

### Альтернативи, що не пройшли
- Використання `requests` + `xml.etree` — зайва складність.
- Парсинг HTML (BeautifulSoup) — нестабільно, залежить від структури сторінки.

### Наслідки
- Система отримує реальні дані, що дозволяє тестувати гіпотези.
- Потрібно додати обробку різних форматів (RSS 2.0, Atom).
- У майбутньому можна легко додати нові джерела, розширивши список URL.

## Майбутні плани
- Додати Twitter API (потребує автентифікації).
- Додати Google News RSS.
- Додати фільтрацію дублікатів за заголовками/посиланнями.
