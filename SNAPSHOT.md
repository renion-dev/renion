## Technical State (Технічний стан)

- **Мова:** Python 3.10+ (asyncio)
- **База даних:** SQLite + aiosqlite
- **Архітектура:** Clean Architecture (domain / application / infrastructure / interfaces)
- **Код:** простий, типізований, з тестами
- **Якість:** pytest (2 passed), ruff (passed), mypy (passed з нотатками)
- **Залежності:** aiosqlite, pytest, ruff, mypy, pytest-asyncio

## Risks (Ризики)

1. **RSS-читач — заглушка.** Потрібна реальна реалізація для роботи з різними форматами.
2. **Немає обробки помилок.** Потрібно додати логування та retry-механізми.
3. **Обмежена масштабованість.** SQLite не підходить для великих навантажень — у майбутньому перехід на PostgreSQL.
4. **Відсутній економічний шар.** Згідно з концепцією RENION, кожна дія має оцінюватися за вартістю/цінністю.
5. **Немає інтерфейсу користувача.** Поки тільки CLI.

---

## Next Steps (Наступні кроки)

1. **Реалізувати реальний RSS-парсер** (замінити заглушку) + додати інші джерела (Twitter API, Reddit).
2. **Інтегрувати LLM** (локальну через Ollama або API) для аналізу текстів та валідації гіпотез.
3. **Додати економічний шар:** оцінка вартості кожної дії, збір телеметрії.
4. **Розробити веб-інтерфейс** (FastAPI + прості шаблони) або Telegram-бота.
5. **Провести тестування з реальними даними** та зібрати зворотний зв'язок.
6. **Підготуватися до релізу v0.2** з розширеним функціоналом.
